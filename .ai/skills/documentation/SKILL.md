# Documentation Skill

Create clear, comprehensive documentation for technical projects.

## Documentation Types

### 1. README.md
```markdown
# Project Name

Brief description of what this project does.

## Features
- Feature 1
- Feature 2

## Installation
\`\`\`bash
npm install project-name
\`\`\`

## Usage
\`\`\`javascript
import { thing } from 'project-name';
\`\`\`

## Configuration
| Variable | Description | Default |
|----------|-------------|---------|
| API_KEY  | Your API key | - |

## Contributing
Guidelines for contributors.

## License
MIT
```

### 2. API Documentation
- Endpoint path and method
- Request parameters (query, body, headers)
- Response format with examples
- Error codes and meanings
- Rate limits if applicable

### 3. Code Comments
- Use JSDoc/docstrings for functions
- Explain WHY, not WHAT (code shows what)
- Document edge cases and gotchas

## Best Practices

1. **Audience**: Write for the intended reader's skill level
2. **Examples**: Include working code examples
3. **Updated**: Keep docs in sync with code
4. **Searchable**: Use clear headings and keywords
5. **Complete**: Cover installation, usage, and troubleshooting
