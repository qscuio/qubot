# Code Explanation Skill

Explain code clearly for different audiences.

## Explanation Levels

### High-Level (For Everyone)
- What problem does this solve?
- What are the inputs and outputs?
- When would you use this?

### Mid-Level (For Developers)
- How does the algorithm work?
- What are the key functions/classes?
- What patterns are used?

### Low-Level (For Experts)
- Why these specific implementations?
- Performance characteristics
- Edge cases and limitations

## Explanation Format

```
## Purpose
[What this code does in one sentence]

## How It Works
1. [Step 1]
2. [Step 2]
3. [Step 3]

## Key Components
- `functionName`: [purpose]
- `ClassName`: [purpose]

## Example
[Simple usage example]

## Notes
- [Important considerations]
```

## Tips

- Use analogies for complex concepts
- Walk through with example data
- Highlight the "why" not just "what"
- Note any non-obvious behavior
- Mention related concepts for learning
